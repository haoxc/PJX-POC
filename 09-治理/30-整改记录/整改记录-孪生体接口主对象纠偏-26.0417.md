# 孪生体接口主对象纠偏整改记录

## 1. 整改依据

本轮整改依据 [[../20-决策记录/ADR-0001-孪生体作为场景构建最小管理单元]]。

核心结论：数字孪生接口治理试点模块应以孪生体为场景搭建最小管理单元，资源、资产、贴图、动画、材质、脚本和 UI 覆盖物仅作为孪生体的依赖引用。

## 2. 整改范围

| 范围 | 结果 |
| --- | --- |
| 治理目录 | 新增 `09-治理` 入口、决策记录索引和整改记录索引 |
| 方向决策 | 新增 ADR-0001，记录从资源口径纠偏为孪生体口径的方向性决策 |
| 管理索引 | 新增 DEC-0004，管理视角保留决策摘要并链接 ADR |
| 阶段规划 | 将规划生命周期别名和关联模块改为孪生体场景构建统一接口 |
| 模块目录 | 将试点模块调整为 `20-模块/孪生体场景构建统一接口` |
| 概要设计 | 将核心对象调整为 `TwinSceneBuildProfile`、`TwinEntity`、`TwinDataBinding`、`TwinPresentationRef` 和 `TwinBuildState` |
| 项目规范 | 补充 `09-治理` 与 `10-管理` 的边界，以及概念范畴不清时先做认知对齐的规则 |

## 3. 整改后口径

- 主对象：孪生体。
- 主模块：孪生体场景构建统一接口。
- 主接口：面向场景自动化构建的孪生体集合查询与构建前校验。
- 资源定位：孪生体的依赖引用，不作为当前模块的治理中心。
- 治理证据：ADR、整改记录和后续审计证据进入 `09-治理`。

## 4. 审计结论

活动文档、生命周期别名、模块入口和设计对象已完成纠偏。

“场景资源统一接口”只保留在 ADR 的历史背景中，用于说明本次方向调整来源，不再作为活动模块、别名或链接入口。

## 5. 遗留事项

- 后续如需要统一管理贴图、动画、材质、模型文件和脚本，应另行评估是否建立“场景表现资源管理”模块。
- 孪生体唯一标识来源、类型体系和数据绑定维护责任仍需在下一轮设计中确认。
