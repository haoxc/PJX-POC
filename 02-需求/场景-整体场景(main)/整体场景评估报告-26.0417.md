## 2. 总体结论

当前整体场景具备厂区级空间底座，能够支撑 POC 的全景开场和区域定位。

主要问题是环境范围与数字孪生建设范围的视觉层级不够清晰。外部环境信息量较大，容易稀释客户对核心建设区域的注意力；孪生区域已经具备主体建筑和设备基础，但还缺少客户识别、工艺表达、数据叠加和系统先进性展示。

POC 阶段不建议继续平均提升全场景精度。应采用“环境做衬托、孪生区域做主角、局部设备做亮点”的策略。

## 3. 范围关系判断

| 范围    | 当前作用             | 后续处理原则           |
| ----- | ---------------- | ---------------- |
| 外部环境  | 提供城市、厂区、周边空间背景   | 保持完整即可，降低视觉权重    |
| 孪生区域  | 本次数字孪生建设主体       | 重点优化，承担 POC 主要表达 |
| 主厂房   | 空间体量最大，具备项目识别潜力  | 增强品牌、标题和入口识别     |
| 工艺设备区 | 包含罐体、构筑物、管线等核心对象 | 强化设备细节、工艺逻辑和数据表达 |
| 局部设备  | 例如厌氧罐            | 用作系统能力展示案例       |

## 4. 环境场景评估

| 评估项    | 当前表现              | 判断              |
| ------ | ----------------- | --------------- |
| 空间尺度   | 环境范围大，周边建筑丰富      | 有利于体现项目位于真实工业园区 |
| 背景完整度  | 远景、道路、厂房、绿化较完整    | 可作为全景开场背景       |
| 视觉干扰   | 周边建筑数量多，明度和体量接近主体 | 会削弱孪生区域的视觉聚焦    |
| 远景处理   | 远处有雾化和简化效果        | 方向正确，但主体也偏灰     |
| POC 价值 | 适合建立空间，但不适合长时间停留  | 应控制在 5-8 秒内     |

环境场景的主要价值是“交代位置和规模”，不是展示系统能力。外部环境应弱化处理，避免成为 POC 的视觉主角。

## 5. 孪生区域评估

| 评估项  | 当前表现                   | 判断          |
| ---- | ---------------------- | ----------- |
| 主体建筑 | 主厂房体量明确，占据核心画面         | 可作为项目主视觉锚点  |
| 工艺设施 | 罐体、构筑物、附属区域可见          | 具备拆分局部场景的基础 |
| 品牌识别 | **客户标识不明显**            | 不利于营销归属     |
| 工艺逻辑 | **设备之间关系不清晰**          | 不利于说明环保处理流程 |
| 数据表达 | 未见状态、指标、告警或标签          | 数字孪生属性不足    |
| 场景质感 | 建筑和设备较完整，但整体**偏灰、偏静态** | 需要提升展示片质感   |
|      |                        |             |

孪生区域已经满足“有场景”的基础条件，但还没有达到“可营销、可讲解、可体现先进性”的 POC 要求。

## 6. 视觉传达问题

### 6.1 主次关系不足

环境图中外部厂房数量多，孪生区域在画面中被周边环境包围。观众需要额外判断哪里是项目主体。

建议在全景进入时，用边界高亮、轻量遮罩、区域名称或镜头推进，将孪生区域从环境中明确提取出来。

### 6.2 客户归属弱

当前主厂房和核心设备区没有足够醒目的“宜兴新奇环保”识别信息。

建议在主厂房立面、入口区域、核心设备区或视频标题层增加客户名称、项目名称或 LOGO。客户识别不应只依赖讲解词。

### 6.3 数字孪生感不足

当前画面主要呈现三维建模结果，没有体现数据接入、设备状态、运行参数、管理动作和异常感知。

建议在孪生区域图中叠加 3 类信息：设备标签、关键运行指标、系统能力提示。

### 6.4 演示路径不明确

当前两张图都适合作为镜头素材，但缺少从环境到孪生区域、再到局部设备的叙事路径。

建议建立固定镜头链路：全景环境 -> 孪生区域 -> 主厂房 -> 工艺设备区 -> 厌氧罐局部 -> 数据能力收束。

## 7. 客户诉求匹配评估

| 客户诉求         | 当前满足度 | 问题                    |
| ------------ | ----- | --------------------- |
| 展示项目整体规模     | 中高    | 环境范围足够，但主体不够突出        |
| 展示数字孪生建设范围   | 中     | target 图可表达范围，但需要显性标注 |
| 展示客户品牌和营销价值  | 低     | 客户标识、项目标题和价值表达不足      |
| 展示系统先进性      | 低     | 缺少数据层、状态层和智能化表达证据     |
| 支撑 UE 动画演示视频 | 中     | 场景基础具备，但镜头脚本和动效设计缺失   |
| 支撑 POC 汇报    | 中低    | 需要补充叙事结构、亮点组织和验收口径    |

## 8. POC 展示策略

建议采用三层展示策略：

| 层级 | 展示目的 | 表达方式 |
| --- | --- | --- |
| 环境层 | 建立项目所在空间和厂区尺度 | 5-8 秒全景俯视，环境弱化，镜头推进 |
| 孪生区域层 | 明确建设范围和客户主体 | 区域高亮、标题、LOGO、POI 标签 |
| 能力层 | 展示数字孪生价值 | 数据卡片、设备状态、工艺流向、告警提示 |

该策略的核心是让客户先看清“范围”，再看懂“业务”，最后感知“系统先进性”。

## 9. 推荐镜头结构

| 镜头段 | 时长 | 内容 | 目标 |
| --- | --- | --- | --- |
| 1. 环境全景 | 5-8 秒 | 展示外部环境和园区规模 | 建立真实空间感 |
| 2. 目标区域锁定 | 5-8 秒 | 镜头推进到孪生区域，区域轻量高亮 | 明确建设范围 |
| 3. 主厂房识别 | 5-8 秒 | 展示主厂房、项目标题、客户标识 | 强化客户归属 |
| 4. 工艺区过渡 | 8-12 秒 | 推进到罐区、膜车间或处理构筑物 | 建立业务路径 |
| 5. 局部设备展示 | 15-25 秒 | 进入厌氧罐等重点设备 | 展示工艺和数据能力 |
| 6. 系统能力收束 | 5-8 秒 | 展示指标、状态、能力标题 | 支撑营销表达 |

## 10. 改善优先级

| 优先级 | 改善项 | 说明 |
| --- | --- | --- |
| P0 | 明确孪生区域边界 | target 区域是 POC 主体，必须在视频中被看见 |
| P0 | 强化客户和项目标识 | 增加“宜兴新奇环保”识别，提升营销归属 |
| P0 | 设计全景到局部的镜头链路 | 解决视频叙事缺失 |
| P0 | 选择 1-2 个局部设备做能力展示 | 避免全场景平均表达 |
| P1 | 外部环境降权处理 | 降低饱和度、对比度、细节密度或景深 |
| P1 | 增加数据标签和状态卡片 | 补足数字孪生感 |
| P1 | 优化光影通透度 | 减少整体发灰和雾感 |
| P2 | 优化远景模型一致性 | 提升整体品质，但不是当前 POC 首要项 |

## 11. 验收标准

| 类别 | 验收标准 |
| --- | --- |
| 范围识别 | 观众能在 10 秒内识别数字孪生建设区域 |
| 客户识别 | 画面中能清晰识别宜兴新奇环保项目 |
| 主次关系 | 外部环境不抢占孪生区域视觉注意力 |
| 业务路径 | 镜头能从整体自然过渡到核心工艺设备 |
| 数字孪生感 | 至少出现设备标签、状态数据或运行指标 |
| 先进性表达 | 至少有一处画面证据支撑系统先进性说明 |
| 视频可用性 | 能支撑一段 45-60 秒的 POC 场景演示 |

## 13. 待确认问题

- target 图标定范围是否等于本次 POC 的最终建设范围。
- 客户希望优先展示主厂房、厌氧罐、膜车间还是其他核心区域。
- 是否允许在视频中显性展示区域边界或高亮框。
- 客户品牌标识的使用位置、尺寸、颜色和审批口径。
- 数据卡片字段是否有真实数据来源，是否允许使用演示数据。
- POC 视频目标时长是 45-60 秒，还是 90 秒以内完整展示。
