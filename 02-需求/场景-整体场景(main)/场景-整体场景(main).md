## 2. 范围标定

用户已明确：图片中绿色框标定区域属于数字孪生建设范畴。

该范围应作为后续场景分析、POC 展示、模型优化和局部整改的上位边界。

| 范围类型      | 管理口径                              |
| --------- | --------------------------------- |
| 绿色框内区域    | 数字孪生建设范围，需要支持展示、讲解、重点优化和业务表达      |
| 绿色框外区域    | 环境衬托范围，主要承担背景氛围，不应投入同等精度          |
| 局部设备区     | 在整体范围内按业务价值拆分分析，例如厌氧罐、膜车间、管线、厂房入口 |
| 远景厂区和城市背景 | 弱化处理，避免抢占客户主体项目的视觉注意力             |

## 3. 初步结论

整体场景已经具备厂区级数字孪生底座，但当前更像“完整园区模型截图”，还没有形成面向 POC 的展示叙事。

绿色框范围内需要建立明确主次：主厂房、工艺设备区、关键罐体、道路动线和数据展示点应被组织成一条可讲解的演示路径。

POC 改善重点不是无限提高全场景精度，而是让客户在 10 秒内看清：这是新奇环保的厂区、这是数字孪生建设范围、这是系统能展示和管理的核心对象。

## 4. 整体视觉评估

| 评估项    | 当前表现                   | 问题判断        |
| ------ | ---------------------- | ----------- |
| 空间完整度  | 主厂房、周边厂区、道路、设备区已形成整体布局 | 可支撑全局展示     |
| 主体识别   | 主厂房体量最大，但客户品牌与项目标识不突出  | 客户归属感不足     |
| 核心业务区  | 右侧罐区、左侧构筑物、主厂房附属区域可见   | 缺少视觉引导和业务层级 |
| 背景环境   | 周边厂房和远景城市信息丰富          | 背景容易分散注意力   |
| 光影氛围   | 整体偏灰、偏雾，远景压低但主体不够通透    | 营销质感不足      |
| 数字孪生表达 | 画面未显示数据、标签、状态或系统能力     | 系统先进性不明显    |
| 演示路径   | 单张全景无法体现镜头动线           | 需要补充导览脚本    |

## 5. 建设范围内对象分层

| 层级      | 对象                   | 展示职责             |
| ------- | -------------------- | ---------------- |
| L1 厂区主体 | 主厂房、核心建筑群            | 建立项目规模和客户主体识别    |
| L2 工艺设施 | 厌氧罐、膜车间、处理构筑物、管线     | 展示环保处理工艺和业务逻辑    |
| L3 数据对象 | 设备状态、运行参数、告警、能耗、处理效率 | 体现数字孪生能力         |
| L4 演示辅助 | POI、标签、镜头路径、引导词      | 支撑 POC 讲解和视频节奏   |
| L5 背景衬托 | 周边厂房、远景城市、外围道路       | 提供空间环境，不作为精细表达重点 |

## 6. 客户诉求视角评估

| 客户诉求 | 当前满足度 | 说明 |
| --- | --- | --- |
| 看出项目整体规模 | 中高 | 全景能体现厂区规模 |
| 看出客户主体项目 | 中低 | 缺少醒目的客户标识和项目范围提示 |
| 看出数字孪生系统能力 | 低 | 缺少数据层、状态层和管理动作 |
| 看出环保工艺价值 | 低 | 工艺链路、设备关系和指标表达不足 |
| 适合 POC 营销汇报 | 中低 | 缺少镜头叙事、视觉重点和先进性表达 |

## 7. POC 展示叙事建议

整体场景应承担视频开场和范围建立作用，不宜停留过久。

推荐顺序：

| 阶段 | 时长 | 展示目标 | 表达方式 |
| --- | --- | --- | --- |
| 开场全景 | 5-8 秒 | 建立厂区规模和建设范围 | 俯视推进，绿色框范围可用轻量轮廓或边界高亮 |
| 主体识别 | 5-8 秒 | 明确宜兴新奇环保项目 | 主厂房与核心区域出现项目名、LOGO 或标题 |
| 工艺导入 | 8-12 秒 | 从全厂过渡到核心工艺区 | 镜头推进到厌氧罐、膜车间或关键设备区 |
| 数据叠加 | 8-12 秒 | 表达数字孪生能力 | 弹出运行状态、处理效率、能耗或告警信息 |
| 局部展开 | 15-25 秒 | 进入重点局部场景 | 衔接厌氧罐等局部分析段落 |

## 8. 与局部场景的关系

整体场景负责回答“数字孪生建设范围在哪里、系统覆盖什么区域、客户项目规模如何”。

局部场景负责回答“核心设备如何运行、数据如何呈现、系统先进性如何被看见”。

厌氧罐局部分析应作为整体场景的 L2 工艺设施案例，不应脱离整体范围单独展示。

## 9. 优先级建议

| 优先级 | 改善项 | 原因 |
| --- | --- | --- |
| P0 | 明确绿色框内为数字孪生建设范围 | 建立范围边界，避免分析对象发散 |
| P0 | 强化主厂房和核心区客户标识 | 解决客户归属感和营销识别 |
| P0 | 设计全景到核心工艺区的镜头路径 | 支撑 POC 视频开场 |
| P1 | 弱化远景和周边非核心厂区 | 避免背景抢占视觉注意力 |
| P1 | 增加 POI 或范围标签 | 帮助客户快速理解建设内容 |
| P1 | 增加系统能力数据层 | 补足数字孪生先进性表达 |
| P2 | 提升远景环境一致性 | 改善整体质感，但不是 POC 首要项 |

## 10. 验收标准

| 类别 | 验收标准 |
| --- | --- |
| 范围识别 | 观众能明确绿色框内为数字孪生建设范围 |
| 客户识别 | 观众能明确这是宜兴新奇环保项目 |
| 主次关系 | 主厂房和核心工艺区明显强于背景厂区 |
| 镜头导览 | 全景镜头能自然过渡到一个核心局部场景 |
| 数字孪生感 | 画面出现至少一类数据、状态或系统能力表达 |
| POC 叙事 | 能用 30 秒从全景讲到核心工艺价值 |

## 11. 待确认问题

- 绿色框范围是否等于本次 POC 的全部建设范围，还是仅代表当前截图中的重点展示范围。
- 主厂房、罐区、膜车间、入口区中，哪个区域是 POC 第一优先级。
- 客户是否要求在画面中保留建设范围边界，还是边界只作为内部分析依据。
- 是否允许对绿色框外背景做降饱和、虚化、简模或雾化弱化。
- POC 视频是否需要从整体场景开场，还是直接进入核心设备区。
