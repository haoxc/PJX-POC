# 需求课题初步分析

## 1. 初步课题名称

宜兴新奇环保数字孪生 POC 展示效果与系统先进性表达优化。

## 2. 初步判断

本课题当前不是场景建设或数据接入问题，而是 POC 展示效果与营销表达问题。

现有项目场景和数据接入已经完成，但呈现效果、基于 UE 动画的演示视频、系统先进性展示与阐述尚未满足客户诉求。

初步目标是在 2026-04-20 周一 POC 提交前，明确改善重点，形成可交付、可讲解、能支撑营销诉求的展示方案。

## 3. 当前事实

| 项目   | 当前状态                     |
| ---- | ------------------------ |
| 客户   | 宜兴新奇环保                   |
| 项目类型 | 数字孪生项目                   |
| 场景建设 | 已完成                      |
| 数据接入 | 已完成                      |
| 当前问题 | 呈现效果与 UE 动画演示视频不满足客户营销诉求 |
| 缺失内容 | 系统先进性展示和阐述               |
| 近期节点 | 2026-04-20 周一提交 POC      |

## 4. 技术方案边界

现阶段不优先重做场景和数据接入。

现阶段优先围绕 UE **场景呈现、镜头动画、演示视频、系统亮点表达和 POC** 叙事结构做改善。

如后续客户补充新的功能需求，再判断是否进入数据接口、业务流程或系统架构调整。

## 5. 场景内容构成

| 内容类型    | 当前状态  | 改善重点                 |
| ------- | ----- | -------------------- |
| 数字孪生场景  | 已完成   | 画面质感、空间层次、重点区域突出     |
| 数据接入    | 已完成   | 数据表达方式、指标可读性、业务价值说明  |
| UE 动画演示 | 不满足诉求 | 镜头设计、动画节奏、转场、重点路径    |
| 系统先进性表达 | 缺失    | 技术亮点、业务价值、可视化能力、扩展能力 |

## 6. 后续分析重点

- 客户对“营销效果”的具体判断标准。
- 当前呈现效果不满足诉求的具体原因。
- POC 视频需要覆盖的核心业务场景。
- UE 动画演示的镜头脚本、节奏和重点表达。
- 系统先进性的技术亮点和业务价值阐述。
- POC 交付材料清单和验收口径。

## 7. 优化后的需求课题描述

本课题面向宜兴新奇环保数字孪生 POC，聚焦在已完成场景与数据接入基础上的展示效果提升、UE 动画演示视频优化和系统先进性表达补强。

当前重点不是重构系统功能，而是将已有数字孪生能力转化为客户可感知、可理解、可传播的 POC 展示内容。

后续需要根据客户补充需求，进一步拆解展示改善重点、演示视频脚本、系统亮点话术和 POC 交付边界。

## 8. 一句话定义

本课题是宜兴新奇环保数字孪生 POC 的展示效果、UE 动画视频和系统先进性表达优化。

## 9. 待补充客户需求

后续等待补充客户改善重点。

优先补充以下信息：

- 客户认为当前效果不足的具体点。
- 客户希望 POC 重点展示的业务场景。
- 是否需要正式演示视频、现场演示脚本或汇报材料。
- 视频时长、分辨率、风格和交付格式。
- 系统先进性需要面向管理层、业务部门还是技术团队表达。
