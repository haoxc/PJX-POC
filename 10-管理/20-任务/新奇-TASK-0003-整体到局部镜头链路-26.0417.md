# 新奇 TASK-0003 整体到局部镜头链路

## 1. 任务目标

设计一条稳定、清晰、可执行的镜头链路，使视频从整体环境自然进入数字孪生区域，再进入核心局部设备。

## 2. 任务范围

| 镜头段 | 目标时长 | 目的 |
| --- | --- | --- |
| 环境全景 | 5-8 秒 | 建立空间尺度 |
| 孪生区域锁定 | 5-8 秒 | 明确建设范围 |
| 主厂房识别 | 5-8 秒 | 建立客户主体 |
| 工艺区过渡 | 8-12 秒 | 建立业务路径 |
| 局部设备展示 | 15-20 秒 | 展示设备和数据能力 |
| 系统能力收束 | 5-8 秒 | 支撑先进性表达 |

## 3. 输入材料

| 输入 | 来源 |
| --- | --- |
| POC 视频主线 | [[../10-计划/PLAN-V001-POC可行计划]] |
| 整体场景评估 | [[../../02-需求/场景-整体场景(main)/整体场景评估报告-26.0417]] |
| 会议纪要 | [[会议纪要-新奇POC演示严谨事项-26.0417]] |

## 4. 输出物

| 输出 | 要求 |
| --- | --- |
| 镜头链路表 | 明确每段镜头内容、时长、转场方式和目标 |
| UE 镜头点位 | 至少包含环境全景、target 区域、主厂房、局部设备 |
| 粗剪视频 | 能连续播放，支撑周日评审 |
| 演示脚本草案 | 与镜头顺序一致 |

## 5. 验收标准

| 标准 | 通过条件 |
| --- | --- |
| 路径完整 | 能从环境进入孪生区域，再进入局部设备 |
| 镜头稳定 | 无明显穿模、跳切、甩镜和视角迷失 |
| 节奏清楚 | 每段镜头有明确目的，不停留在无效画面 |
| 可讲解 | 脚本能跟随镜头自然讲解，不需要额外解释路径 |

## 6. 边界说明

本任务只解决 POC 视频的镜头链路。

不处理以下事项：

- 所有功能区的完整漫游。
- 多套镜头风格版本。
- 复杂交互系统开发。
- 完整宣传片剪辑包装。

## 7. 风险与处理

| 风险 | 处理 |
| --- | --- |
| 镜头路径过长 | 压缩为环境、区域、局部三段 |
| 镜头卡顿 | 减少复杂转场和快速移动 |
| 观众看不懂路径 | 增加区域标题或引导词 |
