# 新奇 TASK-0005 POC提交材料

## 1. 任务目标

在 2026-04-20 周一提交前，完成 POC 材料打包，保证材料可打开、可播放、可解释。

## 2. 任务范围

| 交付物      | 要求                   |
| -------- | -------------------- |
| POC 演示视频 | 45-60 秒优先，最低 30-45 秒 |
| 演示脚本     | 镜头顺序、讲解重点、系统亮点一致     |
| 关键截图     | 整体环境、孪生区域、局部设备       |
| 系统亮点说明   | 3-5 条，每条有画面证据        |
| 待确认清单    | 客户待确认和内部待确认分开        |

## 3. 输入材料

| 输入 | 来源 |
| --- | --- |
| POC 计划 | [[../10-计划/PLAN-V001-POC可行计划]] |
| 执行检查清单 | [[POC执行检查清单-26.0417]] |
| 任务台账 | [[20-任务]] |
| 风险台账 | [[../30-风险/30-风险]] |
| 问题台账 | [[../40-问题/40-问题]] |

## 4. 输出物

| 输出   | 要求                    |
| ---- | --------------------- |
| 提交包  | 包含视频、截图、脚本、亮点说明、待确认清单 |
| 文件命名 | 包含项目、POC、日期或版本号       |
| 提交说明 | 简要说明材料内容和待确认问题        |
| 保底材料 | 如果视频失败，提交截图、脚本和短视频片段  |

## 5. 验收标准

| 标准 | 通过条件 |
| --- | --- |
| 视频可播放 | 在提交设备完整播放一遍 |
| 文档可打开 | 所有文档和截图可打开 |
| 内容一致 | 视频、脚本、亮点说明口径一致 |
| 风险透明 | 未确认事项已列入待确认清单 |
| 提交完整 | 不缺视频或替代截图 |

## 6. 边界说明

本任务只负责 POC 提交材料输出。

不处理以下事项：

- 周一新增需求。
- 周一重剪主线。
- 替客户确认未确认事项。
- 制作正式投标级宣传片。

## 7. 一票否决

出现以下任一问题，不建议提交：

- 视频无法播放。
- 客户项目名称错误。
- 画面无法识别数字孪生建设区域。
- 脚本描述了画面中不存在的能力。
- 使用未经确认且可能引发误解的真实业务数据。
