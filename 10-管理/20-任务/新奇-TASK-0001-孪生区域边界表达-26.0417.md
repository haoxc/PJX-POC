# 新奇 TASK-0001 孪生区域边界表达

## 1. 任务目标

明确 POC 视频中数字孪生建设区域的表达方式，让观众在 10 秒内识别建设范围。

## 2. 任务范围

| 项目 | 内容 |
| --- | --- |
| 处理对象 | `img-整体场景(孪生区域-target).jpg` 标定区域 |
| 表达位置 | 视频开场或环境全景进入孪生区域阶段 |
| 表达方式 | 区域高亮、轻量描边、区域标题、POI 标签或镜头推进 |
| 输出用途 | 支撑 POC 视频范围识别和讲解 |

## 3. 输入材料

| 输入 | 来源 |
| --- | --- |
| 整体环境图 | `98-附件/img-整体场景(环境-env).jpg` |
| 孪生区域图 | `98-附件/img-整体场景(孪生区域-target).jpg` |
| 整体场景评估 | [[../../02-需求/场景-整体场景(main)/整体场景评估报告-26.0417]] |
| POC 计划 | [[../10-计划/POC可行计划-26.0417]] |

## 4. 输出物

| 输出 | 要求 |
| --- | --- |
| 边界表达方案 | 明确采用哪种视觉方式表达 target 区域 |
| 视频镜头段 | 5-8 秒，能从环境推进到孪生区域 |
| 截图 | 至少 1 张体现区域范围的截图 |
| 说明句 | 一句话说明该区域为本次数字孪生建设主体 |

## 5. 验收标准

| 标准 | 通过条件 |
| --- | --- |
| 范围可识别 | 第一次观看视频的人员能在 10 秒内指出孪生区域 |
| 视觉不过度 | 高亮或边界不显得像调试框，不破坏展示质感 |
| 叙事可衔接 | 区域锁定后能自然进入主厂房或工艺区 |
| 说明一致 | 视频、截图、脚本对建设范围的表述一致 |

## 6. 边界说明

本任务只解决“建设范围如何被看见”。

不处理以下事项：

- 外部环境精修。
- 所有设备细节优化。
- 真实业务范围审批。
- 复杂数据接入。

## 7. 风险与处理

| 风险 | 处理 |
| --- | --- |
| 客户不希望显性边界 | 改用镜头推进、区域标题或 POI 标签 |
| 高亮影响画面质感 | 使用短时淡入淡出，不长期保留 |
| 范围仍不清晰 | 增加标题“数字孪生建设区域”或轻量遮罩 |
