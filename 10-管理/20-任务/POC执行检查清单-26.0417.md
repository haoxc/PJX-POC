# POC执行检查清单

## 1. 使用规则

本清单用于检查 `POC可行计划-26.0417.md` 的执行结果。

每个阶段结束前必须检查一次。未通过的 P0 项必须处理；P1 项视时间处理；P2 项不阻塞周一提交。

状态取值：`todo`、`doing`、`done`、`blocked`、`skip`。

## 2. 总体口径检查

| 检查项 | 通过标准 | 状态 |
| --- | --- | --- |
| POC 目标一致 | 所有人明确目标是“可汇报、可播放、可解释”，不是系统重构 | todo |
| 范围一致 | 明确环境是衬托，孪生区域是主角，局部设备是亮点 | todo |
| 优先级一致 | P0 优先于 P1，P2 不进入周一前强制排期 | todo |
| 交付物一致 | 视频、脚本、截图、亮点说明、待确认清单均有负责人 | todo |
| 冻结规则一致 | 周日 18:00 后不新增需求，只修阻断问题 | todo |

## 9. 一票否决项

出现以下任一问题，不建议直接提交：

- 视频无法播放。
- 客户项目名称错误。
- 画面无法识别数字孪生建设区域。
- 脚本描述了画面中不存在的能力。
- 使用未经确认且可能引发误解的真实业务数据。
- 提交包缺少视频或替代截图。
