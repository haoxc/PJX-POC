# 项目管理

## 1. 目录定位

`10-管理` 用于管理项目级计划、任务、风险、问题和决策索引。

本目录不存放需求原文、场景分析正文、技术方案正文或 ADR 源记录。

正式治理证据进入 [[../09-治理/09-治理]]。管理目录只保留执行视角下的摘要、状态和来源链接，不把治理目录作为管理子入口。

## 2. 管理入口

| 目录 | 用途 |
| --- | --- |
| [[10-计划/10-计划]] | 项目计划、POC 交付计划、里程碑 |
| [[20-任务/20-任务]] | 任务拆解、执行状态、负责人和截止时间 |
| [[30-风险/30-风险]] | 风险识别、影响、应对策略 |
| [[40-问题/40-问题]] | 当前阻塞、待确认事项、跟进状态 |
| [[60-决策索引/60-决策索引]] | 关键决策摘要和来源索引 |

## 4. 管理规则

- 计划类记录进入 `10-计划`。
- 任务类记录进入 `20-任务`。
- 风险类记录进入 `30-风险`。
- 问题和待确认项进入 `40-问题`。
- 决策索引只记录摘要和链接，不存放 ADR 源记录。
- 正式 ADR、审计证据、评审记录和变更记录进入 `09-治理`。
- 管理文档需要引用治理证据时，只在来源、关联决策或决策索引中链接 `09-治理`，不在管理入口中并列纳入。
