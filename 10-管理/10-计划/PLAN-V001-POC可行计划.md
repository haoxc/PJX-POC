# POC可行计划

## 1. 目标

在 2026-04-20 周一提交 POC 前，完成一版可汇报、可播放、可解释的数字孪生展示方案。

目标不是重做系统，而是在现有场景和数据接入基础上，补齐展示效果、镜头叙事、客户识别和系统先进性表达。

## 2. 交付物

| 交付物      | 要求                                   |
| -------- | ------------------------------------ |
| POC 演示视频 | 45-60 秒，能从整体环境进入数字孪生区域，再进入 1 个核心局部设备 |
| 演示脚本     | 说明镜头顺序、讲解重点、系统亮点                     |
| 关键截图     | 至少 3 张：整体环境、孪生区域、局部设备                |
| 系统亮点说明   | 3-5 条，必须有画面证据支撑                      |
| 待确认清单    | 标记后续需要客户确认的品牌、数据、工艺和范围问题             |
|          |                                      |

## 3. 工作原则

- 只做 POC `必需项`，不做`全场景平均精修`。
- 环境做衬托，孪生区域做主角，局部设备做亮点。
- 周日 18:00 后冻结画面和镜头，只允许修复阻断问题。
- 周一不新增需求，只做导出、检查和提交。

## 4. 优先级范围

| 优先级 | 必做内容 | 不做内容 |
| --- | --- | --- |
| P0 | 孪生区域边界识别、客户标识、镜头链路、局部设备能力展示 | 重做外部环境 |
| P0 | 45-60 秒视频主线 | 多版本风格探索 |
| P1 | 数据标签、状态卡片、环境降权、光影通透度 | 完整数据平台包装 |
| P2 | 远景模型一致性、细节补充 | 周一 POC 前不排入 |

## 6. 视频主线

| 段落 | 时长 | 内容 | 目的 |
| --- | --- | --- | --- |
| 1. 环境全景 | 5-8 秒 | 展示整体外部环境 | 建立项目空间尺度 |
| 2. 孪生区域锁定 | 5-8 秒 | 镜头推进到 target 区域 | 明确数字孪生建设范围 |
| 3. 客户主体识别 | 5-8 秒 | 展示主厂房、项目名称或 LOGO | 建立客户归属 |
| 4. 工艺区过渡 | 8-12 秒 | 进入罐区或处理构筑物 | 建立业务路径 |
| 5. 局部设备展示 | 15-20 秒 | 展示厌氧罐等核心设备 | 展示工艺和设备能力 |
| 6. 系统能力收束 | 5-8 秒 | 叠加指标、状态、能力标题 | 支撑先进性表达 |

## 7. 必须完成项

| 编号 | 事项 | 验收口径 |
| --- | --- | --- |
| M1 | 孪生区域可识别 | 观众能在 10 秒内知道建设范围在哪里 |
| M2 | 客户项目可识别 | 画面出现宜兴新奇环保或项目名称识别 |
| M3 | 镜头路径完整 | 能从环境进入孪生区域，再进入局部设备 |
| M4 | 局部设备有亮点 | 至少一个核心设备段具备环绕、标签或数据表达 |
| M5 | 系统先进性有证据 | 至少 3 个画面元素支撑系统亮点说明 |
| M6 | 视频可交付 | 视频能稳定播放，时长控制在 45-60 秒 |

## 8. 系统亮点表达

| 亮点 | 画面证据 |
| --- | --- |
| 全域可视 | 从外部环境进入数字孪生区域 |
| 重点区域可定位 | target 区域高亮、POI 或区域标签 |
| 设备运行可感知 | 局部设备状态卡片或参数标签 |
| 工艺流程可讲解 | 从主厂房到工艺区再到设备的镜头链路 |
| 运维管理可支撑 | 告警、状态、指标或能力标题收束 |

## 9. 风险控制

| 风险 | 处理策略 |
| --- | --- |
| 时间不足 | 只保 P0：区域识别、客户标识、镜头链路、局部设备段 |
| 数据字段未确认 | 使用演示字段，并标记为待客户确认 |
| LOGO 素材未确认 | 先使用项目名称文字标识，避免阻塞视频 |
| 工艺逻辑未确认 | 不做复杂管线推演，只做基础设备关系表达 |
| 视频渲染失败 | 保底提交关键截图 + 演示脚本 + 短视频片段 |

## 10. 最低可交付版本

若制作时间被压缩，最低交付版本必须包含：

- 30-45 秒视频。
- 1 个整体环境镜头。
- 1 个孪生区域锁定镜头。
- 1 个局部设备展示镜头。
- 1 张系统先进性说明页。
- 1 份待确认问题清单。

## 11. 不纳入本次 POC 的事项

- 全外部环境精修。
- 所有局部设备逐一精修。
- 完整真实数据接口联调。
- 多套视频风格版本。
- 长篇系统架构说明。
- 未经确认的复杂工艺流程动画。
